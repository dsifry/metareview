metareview

Local-first review gates and learning for coding agents: specs, plans, task-done code chunks, epics, PR readiness, and post-merge follow-up.

Quick Start

npm install -g metareview
metareview setup --check
metareview review task-done <task-id-or-path> --base main --evidence /tmp/evidence.txt

In source checkout mode, build first with npm run build or run commands through go run ./cmd/metareview.

Install Paths

Standalone CLI

Use the npm package or source checkout. Run setup --bootstrap-prereqs --dry-run before adding Superpowers, Beads, metaswarm-compatible files, or registries.

Codex

Install the plugin, then use $setup, $review-task-done, $review-epic-ready, $review-pr-ready, and $learn-post-merge.

Claude Code

Install the plugin, then use /setup, /review-task-done, /review-epic-ready, /review-pr-ready, and /learn-post-merge.

Review Lifecycle

artifact
Review specs, plans, decompositions, designs, and documentation before implementation.
task-done
Review each local code chunk before an agent claims it is complete.
epic-ready
Review parent readiness once child tasks are complete and accepted.
pr-ready
Review the branch before push, PR creation, or merge readiness.
post-merge
Extract lessons from merged PRs into durable local knowledge.

Blocking Policy

A blocking finding is current work. Fix it, re-run with --previous-run <run-id>, and do not claim completion until the gate reports PASS or PASS_ADVISORY with zero blockers.

Metaswarm Fit

When metaswarm, Superpowers, and Beads are present, metaswarm remains the lifecycle owner. metareview augments that workflow with deeper review gates and Markdown-first evidence under docs/metareview/.

When used standalone, metareview remains local-first and can create fallback knowledge files until Beads knowledge is available.
